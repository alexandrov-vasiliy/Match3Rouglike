# GemHunterMatch — Архитектура шаблона Match-3

## Обзор

**GemHunterMatch** — это готовый шаблон Match-3 игры от Unity. Игрок свайпает камни (Gem) на доске (Board), собирая совпадения из 3+ одинаковых камней в линию или фигуру. Есть система целей уровня, бонусных камней, препятствий, магазина и расходуемых предметов.

---

## Структура папок

```
Scripts/
├── Authoring/           — Тайлы для редактора уровней (GemPlacerTile, GemSpawner, ObstaclePlacer)
├── BonusGem/            — Бонусные камни (LineRocket, SmallBomb, LargeBomb, ColorClean)
├── BonusItem/           — Расходуемые предметы (BonusItem, BonusGemBonusItem)
├── Editor/              — Кастомный PropertyDrawer для MatchShape
├── ShopItem/            — Товары магазина (ShopItemBonusItem, ShopItemCoin, ShopItemLive)
├── UI/                  — UI логика (UIHandler, MainMenu)
├── Board.cs             — Игровая доска: основная логика Match-3
├── BoardCell.cs         — Данные одной ячейки доски
├── Crate.cs             — Ящик — неподвижный камень с HP
├── GameManager.cs       — Глобальный менеджер (синглтон DontDestroyOnLoad)
├── GameSettings.cs      — Все настройки игры (визуал, звук, бонусы, магазин)
├── Gem.cs               — Базовый класс камня
├── InitLoader.cs        — Загрузчик начальной сцены
├── LevelData.cs         — Данные уровня (цели, кол-во ходов)
├── LevelList.cs         — ScriptableObject со списком уровней/сцен
├── LightManager.cs      — Управление направлением глобального освещения
├── Match.cs             — Данные одного совпадения (матча)
├── Obstacle.cs          — Базовый класс препятствия
├── TieBlocker.cs        — Препятствие-«обвязка»: блокирует ячейку до уничтожения
└── VFXPoolSystem.cs     — Пул VFX-эффектов (VisualEffect)
```

---

## Жизненный цикл игры

```
[Сцена Init] → InitLoader.Awake() → Загрузка сцены Main Menu (scene index 1)
       ↓
[Сцена MainMenu] → MainMenu.Start() → GameManager.MainMenuOpened()
       ↓                                → Показ списка уровней из LevelList
       ↓
[Игрок выбирает уровень] → LevelList.LoadLevel(index)
       ↓
[Сцена уровня] → LevelData.Awake() → GameManager.StartLevel()
       ↓                              → Board.Init() (через DelayedInit корутину)
       ↓                              → UIHandler.Init()
       ↓
[Игровой цикл] → Board.Update() — обработка ввода, физика камней, проверка матчей
       ↓
[Все цели выполнены / ходы закончились] → UIHandler.ShowEnd() → Win / Lose экран
```

## Ключевые системы и их взаимодействие

### 1. GameManager (Синглтон)

**Файл:** `GameManager.cs`

Центральная точка доступа ко всем системам. Создаётся один раз и живёт между сценами (`DontDestroyOnLoad`).

**Содержит:**
- Ссылку на `Board` (находит через `FindFirstObjectByType`)
- `GameSettings` — все настройки
- Ресурсы игрока: `Coins`, `Stars`, `Lives`
- Список `BonusItems` (расходуемые предметы)
- `VFXPoolSystem` — пул визуальных эффектов
- Аудио: два `AudioSource` для музыки (кроссфейд), пул из 16 SFX источников

**Основные методы:**
| Метод | Описание |
|-------|----------|
| `StartLevel()` | Вызывается из `LevelData.Awake()`. Инициализирует доску и UI |
| `MainMenuOpened()` | Сброс при возврате в меню |
| `ChangeCoins()` | Изменение монет + обновление UI |
| `AddBonusItem()` | Добавление расходуемого предмета |
| `ActivateBonusItem()` | Активация бонусного предмета (режим выбора цели) |
| `UseBonusItem()` | Использование предмета на конкретной ячейке |
| `PlaySFX()` | Воспроизведение звука из пула |
| `WinTriggered() / LooseTriggered()` | Эффекты победы/поражения |

---

### 2. Board (Игровая доска)

**Файл:** `Board.cs`

Основной класс игровой логики. Управляет всем игровым циклом Match-3.

**Структура данных:**
- `CellContent: Dictionary<Vector3Int, BoardCell>` — все ячейки доски
- `SpawnerPosition: List<Vector3Int>` — позиции спавнеров новых камней
- `ExistingGems: Gem[]` — массив доступных типов камней

**Игровой цикл (Update):**
```
1. VFXPoolSystem.Update()         — обновление пула эффектов
2. BoardActions.Tick()             — тик всех активных Board-действий (ракеты и т.д.)
3. CheckInput()                    — обработка свайпов/кликов игрока
4. TickSwap()                      — анимация обмена двух камней
5. MoveGems()                      — падение камней (гравитация)
6. DoMatchCheck()                  — проверка совпадений (линии 3+ и фигуры бонусов)
7. MatchTicking()                  — анимация удаления совпавших камней
8. EmptyCheck()                    — заполнение пустых ячеек (падение сверху / спавн)
9. Обработка подсказок (hint)      — показ подсказки при бездействии игрока
10. Проверка конца игры            — если m_FinalStretch и доска стабилизировалась
```

**Ключевые механики:**

| Механика | Описание |
|----------|----------|
| **Свайп** | Игрок свайпает → камни меняются местами → проверка матча → если нет матча, откат |
| **Падение** | Камни падают вниз, с поддержкой диагонального падения (если прямо заблокировано) |
| **Матч** | BFS-поиск связных камней одного типа → поиск линий 3+ → проверка фигур бонусов |
| **Спавн** | Когда пустая ячейка под спавнером — создаётся новый случайный камень |
| **Подсказка** | Если игрок бездействует `InactivityBeforeHint` секунд — подсвечивается возможный ход |
| **FreezeMoveLock** | Блокировка падения (используется ракетами, чтобы камни не падали во время полёта) |

**Статические методы (вызываются Authoring-тайлами):**
- `RegisterCell()` — регистрация ячейки доски
- `RegisterSpawner()` — регистрация спавнера
- `AddObstacle()` — добавление препятствия
- `ChangeLock()` — блокировка/разблокировка ячейки
- `RegisterDeletedCallback()` / `RegisterMatchedCallback()` — колбэки при удалении/совпадении

### 3. BoardCell (Ячейка доски)

**Файл:** `BoardCell.cs`

Простая структура данных для одной ячейки.

| Поле | Описание |
|------|----------|
| `ContainingGem` | Камень, находящийся в ячейке |
| `IncomingGem` | Камень, который сейчас падает в эту ячейку |
| `Obstacle` | Препятствие (если есть) |
| `Locked` | Заблокирована ли ячейка (нельзя двигать/удалять) |

| Свойство | Логика |
|----------|--------|
| `CanFall` | Камень может упасть (пустая или подвижный камень, не залочена) |
| `BlockFall` | Блокирует падение (залочена или неподвижный камень) |
| `CanBeMoved` | Можно свайпнуть (не залочена, есть подвижный камень) |

---

### 4. Gem (Камень)

**Файл:** `Gem.cs`

Базовый класс для всех камней. Является `MonoBehaviour`, создаётся как префаб.

**Состояния (State):**
```
Still → Falling → Bouncing → Still (при приземлении)
                           → Disappearing (при удалении)
```

| Поле | Описание |
|------|----------|
| `GemType` | Тип камня (int). Камни одного типа матчатся |
| `CanMove` | Может ли камень двигаться (false у Crate) |
| `Usable` | Можно ли активировать (true у бонусных камней) |
| `CurrentMatch` | Текущий матч, в котором участвует камень |
| `m_HitPoints` | Очки здоровья (для ящиков) |

**Наследники:**
- `BonusGem` — базовый класс бонусных камней
- `Crate` — ящик (неподвижный, разрушается от соседних матчей)

---

### 5. Match (Совпадение)

**Файл:** `Match.cs`

Данные одного совпадения.

| Поле | Описание |
|------|----------|
| `MatchingGem` | Список позиций совпавших камней |
| `OriginPoint` | Точка, с которой начался матч |
| `DeletionTimer` | Таймер анимации удаления |
| `SpawnedBonus` | Бонусный камень, который спавнится на месте матча |
| `ForcedDeletion` | Принудительное удаление (от бонуса), удаляет и препятствия |
| `DeletedCount` | Счётчик удалённых камней (при 4+ даёт монеты) |

---

### 6. LevelData (Данные уровня)

**Файл:** `LevelData.cs`

Находится в каждой сцене уровня. Содержит цели и лимит ходов.

| Поле | Описание |
|------|----------|
| `MaxMove` | Максимальное количество ходов |
| `Goals: GemGoal[]` | Цели: собрать N камней определённого типа |
| `LowMoveTrigger` | При каком количестве ходов показать предупреждение |

**События:**
- `OnAllGoalFinished` — все цели выполнены → победа
- `OnNoMoveLeft` — ходы закончились → поражение
- `OnGoalChanged` — обновление прогресса цели
- `OnMoveHappened` — совершён ход

### 7. Система бонусных камней

**Иерархия:**
```
Gem
 └── BonusGem (базовый, содержит MatchShape)
      ├── SmallBomb      — взрыв 4 соседних клетки (крест)
      ├── LargeBomb      — взрыв 5×5 область
      ├── LineRocket      — ракета по горизонтали/вертикали (уничтожает всё в линии)
      └── ColorClean      — уничтожает все камни одного цвета на доске
```

**Логика появления:**
1. При матче `Board.DoCheck()` ищет все `BonusGem` из `GameSettings.BonusSettings`
2. Для каждого бонуса проверяются его `MatchShape` — подходит ли форма матча
3. Если подходит — на месте матча спавнится бонусный камень
4. `MatchShape` поддерживает повороты (`CanRotate`) и зеркалирование (`CanMirror`)

**Активация:**
- **Свайп** — если бонусный камень свайпнули, вызывается `Use(swappedGem)`
- **Двойной клик** — вызывается `Use(null)`
- **Цепная реакция** — если бонус попал под другой бонус, вызывается `Use(null)`

---

### 8. Система препятствий

**Иерархия:**
```
MonoBehaviour
 └── Obstacle (абстрактный)
      └── TieBlocker — блокирует ячейку, разрушается при матчах в ней

Gem
 └── Crate — ящик (занимает ячейку, разрушается от соседних матчей)
```

**Obstacle** (не является Gem):
- Размещается поверх ячейки
- Имеет несколько стадий (`LockState`) с разными спрайтами и VFX
- `TieBlocker` — блокирует камень в ячейке, но разрешает матчи; каждый матч снимает слой

**Crate** (является Gem):
- Не двигается (`CanMove = false`)
- Имеет HP, получает урон от соседних матчей через `RegisterDeletedCallback`

---

### 9. Система расходуемых предметов (BonusItem)

```
BonusItem (ScriptableObject, абстрактный)
 └── BonusGemBonusItem — использует BonusGem как расходуемый предмет
```

**Цикл использования:**
1. Игрок покупает предмет в магазине → `GameManager.AddBonusItem()`
2. Нажимает на предмет в нижней панели → `GameManager.ActivateBonusItem()`
3. Кликает на ячейку доски → `GameManager.UseBonusItem()` → `BonusItem.Use(cell)`
4. `BonusGemBonusItem.Use()` — инициализирует временный `BonusGem` и вызывает его `Use()`

### 10. Магазин (Shop)

```
ShopSetting.ShopItem (абстрактный ScriptableObject)
 ├── ShopItemBonusItem — покупка расходуемого предмета
 ├── ShopItemCoin      — покупка монет (демо)
 └── ShopItemLive      — покупка жизней (демо)
```

Товары настраиваются в `GameSettings.ShopSettings.Items`.

### 11. Authoring (Тайлы для редактора уровней)

```
TileBase
 ├── GemPlacerTile    — размещение ячейки доски (+ опционально конкретный камень)
 ├── GemSpawner       — размещение спавнера новых камней
 └── ObstaclePlacer   — размещение препятствия
```

Все три тайла работают через Unity Tilemap:
- В **редакторе** — показывают превью-спрайт для визуального дизайна уровня
- В **рантайме** — вызывают `Board.RegisterCell()` / `Board.RegisterSpawner()` / `Obstacle.Init()`
- Спрайты в рантайме скрываются (тайлы становятся невидимыми)

---

### 12. UI (UIHandler + MainMenu)

**UIHandler** — Singleton на основе UI Toolkit (`UIDocument`). Управляет:
- Верхняя панель: монеты, жизни, звёзды
- Панель целей: иконки камней + счётчик
- Счётчик ходов
- Нижняя панель: расходуемые предметы
- Экран окончания: победа/поражение, кнопки «Заново» / «К уровням»
- Настройки звука (слайдеры громкости)
- Магазин
- Анимации: полёт совпавших камней к целям, анимация персонажа
- Fade In/Out при переходах между сценами
- Debug-меню (F12 в редакторе)

**MainMenu** — главное меню со списком уровней из `LevelList`.

---

### 13. Вспомогательные системы

**VFXPoolSystem** — пул объектов `VisualEffect`:
- Пред-создаёт N экземпляров каждого VFX-префаба
- `PlayInstanceAt()` — воспроизводит VFX в нужной позиции
- Автоматически деактивирует эффекты, когда частицы заканчиваются

**LightManager** — передаёт направление света в глобальный материал.

**LevelList** — `ScriptableObject` со списком сцен уровней. В редакторе работает через `SceneAsset`, в билде — через индексы сцен. Содержит `IPreprocessBuildWithReport` для автоматической синхронизации Build Settings.

## Поток данных при свайпе

```
1. Board.CheckInput()  → Определяет начальную и конечную ячейки свайпа
2. Board.Update()      → m_SwipeQueued = true → обмен IncomingGem между ячейками
3. Board.TickSwap()    → Анимация перемещения двух камней
4. Board.DoCheck()     → BFS поиск связных камней одного типа
   4a. Проверка линий 3+ 
   4b. Проверка MatchShape для бонусных камней
   4c. Если совпадения нет → откат свайпа (SwapStage.Return)
5. Match создаётся     → Камни добавляются в MatchingGem
6. Board.MatchTicking() → Анимация исчезновения, уведомление LevelData.Matched()
7. Board.EmptyCheck()  → Камни сверху падают в пустые ячейки
8. Board.MoveGems()    → Анимация падения + отскока
9. Повторение с п.6    → Каскадные совпадения
10. Стабилизация       → FindAllPossibleMatch() → Обновление подсказок
```
